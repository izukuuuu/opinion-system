# 舆情分析方法论

该 skill 用于把报告生成从“结果罗列”切换成“证据组织 + 机制解释 + 边界控制”的分析写作模式。

先判断现象处在什么阶段，再解释为什么形成、影响如何扩散、哪些结论还不能写满。

## Goal

将报告生成从单轮文案补全提升为带方法论约束的分段研判，优先输出可核验、可复用、可解释的舆情分析文本。

## Reasoning Style

- 先判断事件所处生命周期，再解释传播动力、结构变化与风险迁移。
- 优先识别议程设置、框架竞争、风险传播、情绪放大和平台迁移机制。
- 每一段都要给出主线判断、事实依据和写作边界，不直接堆砌统计结果。
- 结构拆解优先采用“现象-成因-影响”链条，不把多维数据写成孤立清单。

## Language Requirements

- 文本面向高知读者、业务研究人员或传播观察者，重点解释机制、结构与现实含义。
- 坚持事实先行、判断克制、解释充分、表达可读，避免空泛结论与结果拼接。
- 主体、机构和来源判断必须以原始数据源中可验证的字段为边界，不能超范围补主体。
- 证据不足时只能写成待核验项、倾向性判断或观察线索，不能直接上升为事实结论。
- 正文不得暴露系统内部字段名、分析模块名、模型名或工具名。

## Constraints

- 不得把外部检索入口或专家笔记直接写成已验证事实。
- 不得脱离时间线、情感、主题和渠道分布进行空泛总结。
- 结论优先服务于传播研判和处置建议，而不是单纯做数据复述。
- 若引用支持文件，只读取当前任务真正需要的部分，不把整份参考材料原样复述进正文。
