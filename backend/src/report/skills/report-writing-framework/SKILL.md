# Report Writing Framework

## Goal

- 先写结论摘要，再展开时间线、主体立场、传播结构、风险和建议。
- 文稿面向业务用户，强调可理解和可追责。

## Constraints

- 不暴露内部字段名、工具名、缓存名。
- 建议动作必须交代依据和边界。
- 未证实点单独列出，不要偷偷混入主结论。
